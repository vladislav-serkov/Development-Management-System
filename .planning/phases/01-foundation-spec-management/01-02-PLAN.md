       `logger = logging.getLogger(__name__)`

       **`def _get_client() -> anthropic.AsyncAnthropic`:**
       - Return `anthropic.AsyncAnthropic(api_key=settings.anthropic_api_key)`

       **`def _build_document_block(pdf_b64: str, cache: bool = False) -> dict`:**
       - Return dict `{"type": "document", "source": {"type": "base64", "media_type": "application/pdf", "data": pdf_b64}}`
       - If `cache=True`, add `"cache_control": {"type": "ephemeral"}` at the top level of the block

       **`def _log_cache_stats(usage, call_name: str) -> None`:**
       - Log `input_tokens`, `cache_creation_input_tokens`, `cache_read_input_tokens` from usage object using `logger.info`
       - Use `getattr(usage, field, 0)` to safely handle missing fields

       **`def _extract_json_from_text(text: str) -> dict`:**
       - First try: `json.loads(text.strip())`
       - On JSONDecodeError: try regex `r"```(?:json)?\s*\n?([\s\S]+?)\n?\s*```"` to extract content from markdown fences
       - If regex matches: `json.loads(match.group(1).strip())`
       - If nothing works: raise ValueError with first 200 chars of text for debugging
       - This addresses REVIEW CONCERN #5: Markdown fence handling

       **`async def _detect_features(pdf_b64: str, client: anthropic.AsyncAnthropic, model: str) -> FeatureDetectionResult`:**
       - This uses tool_use + manual Pydantic validation (NOT `client.messages.parse()`) — addresses REVIEW CONCERN #1: async parse() uncertainty
       - Build the tool definition:
         ```python
         tool = {
             "name": "detect_features",
             "description": "Extract all features detected in the technical specification",
             "input_schema": FeatureDetectionResult.model_json_schema(),
         }
         ```
       - Call `await client.messages.create(...)` with:
         - `model=model`
         - `max_tokens=4096`
         - `tools=[tool]`
         - `tool_choice={"type": "tool", "name": "detect_features"}`
         - `messages` with single user message containing:
           - `_build_document_block(pdf_b64, cache=False)` — no cache on 1st call
           - Text block: `"This is a technical specification (ТЗ) for a microservice. Identify every distinct feature defined in this document. Each Kafka topic consumer, REST endpoint path, and scheduled task is a separate feature. Extract name, type, confidence (0.0-1.0), one-line summary, and dependency names for each feature."`
       - Parse response: find content block with `type == "tool_use"` in `response.content`
       - Validate: `FeatureDetectionResult.model_validate(tool_block.input)`
       - Log cache stats: `_log_cache_stats(response.usage, "detect_features")`
       - Return the validated result

       **`async def _extract_single_feature_logic(pdf_b64: str, feature: DetectedFeature, client: anthropic.AsyncAnthropic, model: str) -> dict`:**
       - This extracts business logic for ONE feature. Called in parallel for all features.
       - Call `await client.messages.create(...)` with:
         - `model=model`
         - `max_tokens=8192`
         - `messages` with single user message containing:
           - `_build_document_block(pdf_b64, cache=True)` — cache=True for prompt caching
           - Text block with prompt:
             ```
             f"Focus on the feature '{feature.name}' (type: {feature.type.value}).\n\n"
             "Return a JSON object with the complete business logic for this feature, "
             "optimized for an LLM coding agent to generate implementation code.\n\n"
             "Include these aspects where applicable:\n"
             "- processing_steps: ordered list of what the feature does\n"
             "- input_schema: message/request format\n"
             "- output_schema: response format (if any)\n"
             "- error_handling: rules for different error scenarios\n"
             "- external_api_calls: any HTTP calls to other services\n"
             "- database_operations: tables read/written\n"
             "- cache_operations: Redis/cache interactions\n"
             "- business_rules: validation, conditions, edge cases\n\n"
             "Return ONLY the raw JSON object. No markdown fencing, no explanation text."
             ```
       - Parse response: `_extract_json_from_text(response.content[0].text)`
       - Log cache stats: `_log_cache_stats(response.usage, f"business_logic:{feature.name}")`
       - Return the parsed dict

       **`async def _extract_all_business_logic(pdf_b64: str, features: list[DetectedFeature], client: anthropic.AsyncAnthropic, model: str) -> list[tuple[DetectedFeature, dict | None, str | None]]`:**
       - This runs all feature extractions IN PARALLEL using asyncio.gather — addresses REVIEW CONCERN #4
       - For each feature, create a coroutine that:
         - Calls `_extract_single_feature_logic`
         - On success: returns `(feature, result_dict, None)`
         - On Exception: logs the error, returns `(feature, None, str(e))`
       - Use `asyncio.gather(*tasks)` to run all concurrently
       - Return list of (feature, business_logic_or_None, error_or_None) tuples

       **`async def run_extraction_pipeline(filename: str, pdf_bytes: bytes, session: AsyncSession) -> DocumentResponse`:**
       - Encode PDF: `pdf_b64 = base64.standard_b64encode(pdf_bytes).decode("utf-8")`
       - Create Document: `doc = Document(filename=filename, pdf_size_bytes=len(pdf_bytes), status="processing")`
       - `session.add(doc)`, `await session.flush()` to get doc.id
       - Create client: `client = _get_client()`
       - model: `settings.claude_model`
       - **Phase 1: Feature detection**
         - `doc.status = "processing"`
         - Call `detection_result = await _detect_features(pdf_b64, client, model)`
         - Create Feature records for each detected feature: status="detected", no business_logic yet
         - `doc.feature_count = len(detection_result.features)`
         - `doc.status = "extracting"`
         - `await session.flush()`
       - **Phase 2: Business logic extraction (parallel)**
         - Call `results = await _extract_all_business_logic(pdf_b64, detection_result.features, client, model)`
         - For each `(feature_schema, bl_dict, error)` in results:
           - Find the matching Feature ORM record by name
           - If bl_dict is not None: set `feature_orm.business_logic = json.dumps(bl_dict, ensure_ascii=False)`, `feature_orm.status = "done"`, `feature_orm.extracted_at = datetime.utcnow()`
           - If error is not None: set `feature_orm.status = "error"`, `feature_orm.error_message = error`
       - **Final status** — addresses REVIEW CONCERN #2: Partial failure handling
         - Count features with status "done" vs "error"
         - If all done: `doc.status = "done"`
         - If all error: `doc.status = "error"`, `doc.error_message = "All feature extractions failed"`
         - If mixed: `doc.status = "partial"`, `doc.error_message = f"{error_count} of {total} features failed extraction"`
         - `await session.commit()`
       - Build and return DocumentResponse (use feature_to_response helper for each feature)
       - **Error handling (outer try/except):**
         - If detection itself fails (1st call), set `doc.status = "error"`, `doc.error_message = str(e)`, commit, and re-raise as an HTTPException or let it propagate
         - Wrap the entire function in try/except: on any unexpected error after document creation, ensure doc gets error status and session is committed before re-raising
