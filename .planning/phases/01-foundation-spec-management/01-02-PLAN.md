    1. Create `app/services/extraction.py` with the two-call Claude pipeline:

       **Imports:** `base64`, `json`, `re`, `logging`, `anthropic` (use `anthropic.AsyncAnthropic` for async), `AsyncSession` from sqlalchemy, models and schemas from app.

       **`async def run_extraction_pipeline(filename: str, pdf_bytes: bytes, session: AsyncSession) -> DocumentResponse`:**
       - Create Document record with status="processing", add to session, flush to get id
       - Call `_detect_features(pdf_b64, client, model)` for the 1st structured call
       - For each detected feature, call `_extract_business_logic(pdf_b64, feature, client, model)` for the 2nd cached call
       - Create Feature records from results, add to session
       - Update Document status to "done"
       - Commit session
       - Return DocumentResponse built from the document + features
       - Wrap in try/except: on error, set document.status="error", document.error_message=str(e), commit, re-raise

       **`_get_client() -> anthropic.AsyncAnthropic`:**
       - Return `anthropic.AsyncAnthropic(api_key=settings.anthropic_api_key)`

       **`_build_document_block(pdf_b64: str, cache: bool = False) -> dict`:**
       - Return dict with `type: "document"`, `source: {type: "base64", media_type: "application/pdf", data: pdf_b64}`
       - If `cache=True`, add `cache_control: {type: "ephemeral"}`

       **`async def _detect_features(pdf_b64: str, client: anthropic.AsyncAnthropic, model: str) -> FeatureDetectionResult`:**
       - Use `await client.messages.create()` with tool_use approach since async `.parse()` availability is uncertain:
         - Define a tool with name `"detect_features"` and `input_schema` from `FeatureDetectionResult.model_json_schema()`
         - Set `tool_choice={"type": "tool", "name": "detect_features"}` to force tool use
         - System prompt: "You are analyzing a technical specification (ТЗ) for a microservice. Identify every distinct feature. Each Kafka topic consumer, REST endpoint path, and scheduled task is a separate feature."
         - User message: document block (no cache on 1st call) + text instruction: "Analyze this technical specification. Identify all features — each Kafka topic consumer, REST endpoint, and scheduled task is a separate feature. Extract name, type, confidence (0.0-1.0), one-line summary, and dependency names for each."
         - Parse the tool_use content block to get the JSON, validate with `FeatureDetectionResult.model_validate(tool_input)`
       - Log cache stats from response.usage: `input_tokens`, `cache_creation_input_tokens`, `cache_read_input_tokens`

       **`async def _extract_business_logic(pdf_b64: str, feature: DetectedFeature, client: anthropic.AsyncAnthropic, model: str) -> dict`:**
       - Use `await client.messages.create()` with:
         - Document block WITH `cache_control: {type: "ephemeral"}`
         - Text prompt: f"Focus on the feature '{feature.name}' (type: {feature.type.value}). Return a JSON object with the complete business logic for this feature, optimized for an LLM coding agent. Include: processing steps, message/request/response schemas, error handling rules, external API calls, database operations, cache operations, business rules, and edge cases. Return ONLY raw JSON, no markdown fencing, no explanation."
         - max_tokens=8192
       - Parse response.content[0].text as JSON
       - Fallback: if JSONDecodeError, try regex `r"```(?:json)?\s*([\s\S]+?)```"` to extract fenced JSON
       - Log cache stats (expect cache_read_input_tokens > 0 on 2nd+ feature)

       **`def _log_cache_stats(usage, call_name: str)`:**
       - Log input_tokens, cache_creation_input_tokens, cache_read_input_tokens using `logger.info`

       **Helper to build Feature ORM from DetectedFeature + business_logic dict:**
       - `dependencies_json` = `json.dumps(feature.dependencies)`
       - `business_logic` = `json.dumps(business_logic_dict, ensure_ascii=False)`

    2. Create `app/routers/documents.py`:

       **Router:** `APIRouter(prefix="/documents", tags=["documents"])`

       **POST `/upload`:**
       - Params: `file: UploadFile = File(...)`, `session: AsyncSession = Depends(get_session)`
       - Validate content_type == "application/pdf", else raise HTTPException(400, "Only PDF files are accepted")
       - Validate file size <= settings.max_pdf_size_mb * 1024 * 1024, else raise HTTPException(413, f"PDF exceeds {settings.max_pdf_size_mb}MB limit")
       - Read file bytes: `contents = await file.read()`
       - Call `await run_extraction_pipeline(filename=file.filename, pdf_bytes=contents, session=session)`
       - Return result (DocumentResponse)

       **GET `/`:**
       - List all documents with features (for debugging/verification)
       - Query Document with selectinload(Document.features), return list[DocumentResponse]

       **GET `/{document_id}`:**
       - Get single document by id with features
       - 404 if not found

    3. Update `app/main.py`:
       - Add `from app.routers.documents import router as documents_router`
       - Add `app.include_router(documents_router)` after CORS middleware
