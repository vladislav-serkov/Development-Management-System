# Milestone v1.0: MVP

**Status:** SHIPPED 2026-03-28
**Phases:** 1-4
**Total Plans:** 9

## Overview

Extract Agent transforms PDF technical specifications into structured `.context/` folders for LLM coding agents. v1.0 delivers the complete pipeline: PDF upload, multi-pass Claude extraction, web UI for viewing and inline editing of all artifacts, and filesystem export.

## Phases

### Phase 1: Foundation + PDF Processing
**Goal**: Users can upload a PDF and the system detects all features (Kafka consumers, REST endpoints, scheduled tasks) with correct type classification
**Depends on**: Nothing (first phase)
**Requirements**: PDF-01, PDF-02, PDF-03, PDF-04, INFR-01, INFR-02, INFR-03
**Plans**: 2 plans

Plans:
- [x] 01-01-PLAN.md — Project scaffold with FastAPI, SQLite, config, ORM models with constraints, and Pydantic schemas with status enums
- [x] 01-02-PLAN.md — Two-call Claude extraction pipeline with parallel extraction, partial failure handling, and integration tests

### Phase 2: Extraction Pipeline
**Goal**: For any uploaded PDF, the system produces a complete and correct .context/ folder with business logic, shared dependency registries, and identified gaps
**Depends on**: Phase 1
**Requirements**: EXTR-01, EXTR-02, EXTR-03, EXTR-04, EXTR-05, EXTR-06, EXTR-07, INFR-04, INFR-05
**Plans**: 2 plans

Plans:

- [x] 02-02-PLAN.md — .context/ export service with additive registry merging and POST endpoint

### Phase 3: Web UI - Viewing
**Goal**: Users can browse extracted context through a web interface with real-time extraction progress and one-click export
**Depends on**: Phase 2
**Requirements**: UI-01, UI-02, UI-03, UI-08, UI-09
**Plans**: 3 plans

Plans:

### Phase 4: Web UI - Editing

## Milestone Summary

**Key Decisions:**
- tool_use with manual Pydantic validation instead of client.messages.parse() for reliability
- Free-text response for 3rd Claude call (complex nested output more reliable as free text)
- Prompt caching on concatenated business-logic context block
- SSE via Starlette StreamingResponse + asyncio.sleep(1) polling
- Two-tier structured data: structured_logic (schema-validated) for UI, business_logic (free-form) for coding agent
- Zustand selectedDocumentId for page routing (no react-router needed)
- CodeMirror for JSON/Markdown editing (~300KB vs Monaco ~5-10MB)

**Issues Resolved:**
- None — clean execution across all phases

**Technical Debt Incurred:**
- greenlet not in pyproject.toml dependencies
- Extraction prompts need tuning on more sample PDFs
- REQUIREMENTS.md traceability table not auto-updated during execution
