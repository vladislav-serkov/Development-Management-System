## What Was Built

### Backend
- **app/schemas/bugs.py**: `BugStep`, `BugReportResult` (Claude tool output), `BugItem` (stored), `BugGenerateRequest`, `BugPatchRequest`
- **app/services/bugs.py**: `generate_bug_report()` — single Claude call with `create_bug_report` tool, builds prompt from test case data + analyst comment, returns structured bug dict
- **app/routers/bugs.py**: `POST /generate`, `GET /`, `PATCH /{index}`, `DELETE /{index}` — follows test_cases.py error handling patterns
- **app/storage.py**: `get_bugs()`, `save_bugs()` — reads/writes `features/{name}/bugs.json`; `save_feature` and `update_feature` updated to handle `bugs` key with `bug_count` sync to `feature.json`
- **app/config.py**: `bugs_model: str = "claude-sonnet-4-6"` added
- **app/main.py**: `bugs_router` registered

### Frontend
- **BugItem/BugsResponse/BugStatus** types added to `api.ts`, `bug_count?: number` added to `FeatureResponse`
- **bugs.ts** API client: `fetchBugs`, `generateBug`, `patchBug`, `deleteBug`
- **useBugs.ts** hooks: `useFeatureBugs`, `useGenerateBug` (invalidates bugs + features queries), `usePatchBug`, `useDeleteBug`
- **BugsView.tsx**: card-based list with status stripes (red=open, amber=fixed, green=verified), Russian labels ("открыт"/"исправлен"/"проверен"), expandable cards showing preconditions/steps/expected/actual, action buttons per status
- **TestCasesView.tsx**: "Баг" button now opens inline form with optional comment textarea + "Создать баг-репорт" submit; shows spinner during generation; on success: marks test case as "edited", switches to Bugs tab
- **ProjectPage.tsx**: Bugs tab added after Tests, Bugs sidebar sub-item with `Bug` icon and count
