## What Was Built

Three fixes to `app/services/test_cases.py`:

### A. `_get_tables_from_dep(dep_data)` — new helper

Extracts table info list from a dependency entry. Handles two formats:
- **Flat format** (real data): `enriched_data` IS the table (`DbTableEnrichment.model_dump()` — has `table_name` key directly)
- **Batch format** (future-proof): `enriched_data` has a `tables` list

The old `_build_fk_tree` called `enriched.get("tables", [])` which always returned `[]` because real data is flat. FK tree always returned `{}`.

### B. `_expand_fk_parents(enriched_deps, flat_deps)` — new function

Recursively finds and includes FK parent tables from `flat_deps` not present in the feature's `enriched_deps`. Uses a `while True` loop that terminates when no new parents are found (stable). Example: feature uses `product_return` (FK -> `product`), but `product` isn't in `used_dependencies` — this function adds it so the FK tree can compute correct INSERT order.

### C. `_build_fk_tree` inner loop rewritten

Replaced the broken `for table in enriched.get("tables", []):` with `for table in _get_tables_from_dep(dep_data):`. Added INFO logging for tables, FK edges, DELETE order, and INSERT order.

### D. `run_test_cases_pipeline` wiring

Added call to `_expand_fk_parents(enriched_deps, flat_deps)` after building initial `enriched_deps`, before passing to `_build_shared_context`.

### E. Removed `thinking` from both Claude calls

Both `_call_plan_phase` and `_call_detail_phase` used `tool_choice={"type": "tool", "name": tool_name}` which forces a specific tool. Anthropic API rejects `thinking` when `tool_choice` forces tool use. Removed `thinking={"type": "enabled", "budget_tokens": N}` from both calls. Also cleaned up stale `max_tokens > budget_tokens` comments and the stale docstring note about thinking.
